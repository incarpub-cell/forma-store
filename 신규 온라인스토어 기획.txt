전체 개념
스토어가 API 서버 역할을 합니다. 어떤 외부 시스템이든 API Key만 있으면 상품을 밀어 넣을 수 있는 구조예요.

핵심 API 설계
POST   /api/v1/products          # 상품 단건 등록
POST   /api/v1/products/bulk     # 상품 대량 등록
PUT    /api/v1/products/:id      # 상품 전체 수정
PATCH  /api/v1/products/:id/price   # 가격만 수정
POST   /api/v1/inventory/sync    # 재고 일괄 동기화
DELETE /api/v1/products/:id      # 상품 삭제
인증 방식
클라이언트(ERP, 스크립트, 파트너사 등)는 요청 헤더에 API Key를 담아 보내고, 서버에서 검증합니다.

Authorization: Bearer sk-live-xxxxxxxxxxxx
Key별로 권한(읽기전용, 상품등록, 전체관리)과 Rate Limit을 다르게 설정할 수 있어요.

업로드 주체 예시
